# Триумф — система управления тренажёрным залом

Веб- и консольная система на Python для учёта в зале **«Триумф»**:
- участников (клиентов);
- тренеров;
- тарифов и абонементов;
- посещаемости;
- расписания групповых занятий.

## Возможности

1. Добавление и просмотр участников.
2. Добавление и просмотр тренеров.
3. Создание тарифов (срок и цена).
4. Выдача абонемента с расчётом даты окончания.
5. Регистрация визитов (с тренером или без).
6. Отчёт по посещаемости за период.
7. Расписание занятий на неделю.

## Запуск консольной версии

Требуется Python 3.10+ (подойдёт и более ранний Python 3, если есть `sqlite3` в стандартной библиотеке).

```bash
python3 app.py
```

После первого запуска будет создана база данных `triumph_gym.db` в текущей папке.

## Запуск web-версии (Flask)

По умолчанию: PostgreSQL, база **`fitness_db`**, пользователь = **имя вашего Mac** (без пароля, типично для Homebrew).

**Не путайте с входом на сайт:** `admin` / `admin123` — только для страницы `/login`, в PostgreSQL роли `admin` обычно нет.

Если задавали `DATABASE_URL` с `admin` — сбросьте и запустите снова:

```bash
unset DATABASE_URL
python3 web_app.py
```

Свой пользователь PostgreSQL:

```bash
export DATABASE_URL=postgresql://ваш_логин_pg@localhost:5432/fitness_db
```

При первом запуске `web_app.py` таблицы создаются автоматически, если их ещё нет.

1) Установить зависимости:

```bash
python3 -m pip install -r requirements.txt
```

2) Запустить сервер:

```bash
python3 web_app.py
```

3) Открыть в браузере:

- **Персонал (админка):** `http://127.0.0.1:5001/`
- **Участники (мобильный кабинет):** `http://127.0.0.1:5001/app/`

Вход участника: телефон из базы + PIN (по умолчанию — **последние 4 цифры** номера).

## Админка

- Страница входа: `http://127.0.0.1:5000/login`
- Логин/пароль по умолчанию: `admin / admin123`

Можно переопределить через переменные окружения:

```bash
export TRIUMPH_ADMIN_LOGIN=superadmin
export TRIUMPH_ADMIN_PASSWORD=strong_password
export FLASK_SECRET_KEY=your_secret
python3 web_app.py
```

Переменные `FITNESS_ADMIN_LOGIN` и `FITNESS_ADMIN_PASSWORD` по-прежнему поддерживаются для совместимости.
